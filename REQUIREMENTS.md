# 药品出入库系统需求分析文档

## 1. 项目概述
本项目旨在开发一套药品出入库管理系统，用于规范医务室的药品管理流程，实现患者就诊记录、药品发放、库存统计及费用核算的数字化管理。

## 2. 技术架构
- **客户端 (Client)**: Python (建议使用 PyQt 或 Tkinter 开发桌面客户端)
- **后端 (Backend)**: Java Spring Boot
- **数据库 (Database)**: MySQL

## 3. 用户角色与权限
系统包含三种角色：
1. **管理员**: 负责系统基础数据维护和人员管理。
2. **医师**: 负责患者就诊登记、处方开具以及药品信息维护。
3. **药师**: 负责审核处方、发药、药品信息维护以及库存盘点。

## 4. 功能模块详细需求

### 4.1 登录系统模块
- **功能描述**: 支持不同角色用户凭账号密码登录。
- **权限控制**: 根据用户角色（管理员、医师、药师）展示相应的操作界面和菜单。

### 4.2 管理员管理模块
- **医生管理**:
  - 查询医生列表。
  - 添加新医生（设置初始账号密码）。
  - 修改医生信息（含重置密码）。
  - 账号状态管理（停用/启用）。
- **日志查询**:
  - 查询医生的操作日志（如登录时间、修改处方记录等），保障数据可追溯。

### 4.3 药品信息管理模块 (医师 & 药师)
- **权限**: 医师和药师均可操作。
- **功能**:
  - **查询**: 支持按药品名称、ID等条件查询药品信息。
  - **新增**: 录入新药品（名称、规格、单位、价格等）。
  - **修改**: 更新药品属性。
  - **删除**: 删除或停用药品。

### 4.4 患者就诊模块 (医师工作站)
- **就诊记录**:
  - 录入信息：姓名、性别、年龄、就诊日期、临床诊断（从下拉列表选择）、药品信息（选择药品ID、输入数量）、更新日期。
  - **诊断类型管理**: 医师可在录入界面新增诊断类型。
- **操作流程**:
  - **填写**: 医师填写就诊信息。
  - **提交**: 提交表单后，状态变为“待发药”，医师此时**无法**点击发药，也无法再次修改（除非被退回）。
  - **修改**: 仅当药师退回处方时，医师可对记录进行修改并再次提交。

### 4.5 药品发放与审核模块 (药师工作站)
- **待办列表**:
  - 实时展示“待发药”的就诊记录。
  - **提示音**: 当有新提交的就诊记录时，系统播放提示音提醒药师。
- **发药操作**:
  - 查看处方详情。
  - **确认发药**: 核对无误后，点击“发药”，扣减库存，状态更新为“已完成”。
  - **退回处理**: 若发现问题（如数量错误、配伍禁忌等），点击“退回”，填写退回原因。记录状态变更为“已退回”，流转回医师端。
- **权限限制**: 药师**不能**录入或修改患者就诊信息。

### 4.6 药品购进与使用统计模块 (月度报表)
- **统计周期**: 按月份统计。
- **默认视图**: 默认显示当前月份的数据。
- **历史查询**: 提供月份选择器（如：2023年10月），点击跳转查询对应月份的历史数据。
- **数据项**:
  - 药品基础信息：药品名称、规格、单位、价格。
  - 期初数据：期初数量、期初金额。
  - 本月购进：购进数量、购进金额。
  - 本月使用：使用数量、使用金额。
  - 期末盘点：期末剩余数量、实际盘点数、盘点金额、备注（盈亏说明）。
- **计算公式**:
  - 期末剩余数量 = 期初数量 + 本月购进 - 本月使用

### 4.7 医务室就诊人数及消耗统计模块 (月度报表)
- **统计周期**: 按月份统计。
- **默认视图**: 默认显示当前月份的数据。
- **历史查询**: 提供月份选择器，支持查看历史月份统计。
- **数据项**:
  - 日期。
  - 就诊人数：当月总就诊人次。
  - 诊治分类：按临床诊断分类统计人数。
  - 药品消耗费用：该月总药品成本。
  - 人均费用：药品消耗费用 / 就诊人数。

### 4.8 库存盘点模块 (药师工作站)
- **权限**: 仅**药师**可操作。
- **触发时间**: 每月月末（如 30 号或 31 号）。
- **盘点流程**:
  1. **任务生成**: 系统自动展示当月所有药品的列表，显示当前系统计算的“理论库存”。
  2. **数据录入**: 药师逐一录入每个药品的“实际盘点数量”和“盘点金额”。
  3. **核对**: 提交后，后端自动对比“理论库存”与“实际盘点数量”，记录盘点结果（盘盈/盘亏/正常）。
- **逾期处理**:
  - 若月末未完成盘点，跨月后（次月1日及以后），系统将该未完成的盘点任务列入“未完成列表”。
  - 药师必须在“未完成列表”中补录上月的盘点数据，才可进行当月的其他操作或清除提醒。

## 5. 业务流程图 (文字描述)

1. **药品维护**: 医师/药师 -> 新增或修改药品库 -> 更新数据库。
2. **就诊/开方**: 医师登录 -> 填写患者信息 & 选择诊断类型(支持新增) & 选择药品 -> 提交 -> (生成待发药记录)
3. **审核/发药**: 
   - 药师登录 -> 收到新消息提醒 -> 查看待发药列表
   - **分支 A (通过)**: 药师核对无误 -> 点击“发药” -> 系统扣库存 -> 流程结束。
   - **分支 B (驳回)**: 药师发现问题 -> 点击“退回” (附注原因) -> 记录返回医师 -> 医师修改 -> 再次提交 -> 回到审核步骤。
4. **月末盘点**: 
   - 月末到达 -> 系统生成盘点任务 -> 药师录入实盘数 -> 提交核对。
   - 若逾期 -> 进入未完成列表 -> 药师补录。

## 6. 数据库设计建议 (核心表)

- **User (用户表)**: id, username, password, role, status
- **Drug (药品表)**: id, name, spec, unit, price, stock_quantity
- **DiagnosisType (诊断类型表)**: id, diagnosis_name, remark
- **PurchaseDetail (购进明细表)**: id, drug_id, quantity (当月购进数量), unit, price, total_amount (购进金额), purchase_date
- **PatientVisit (就诊记录表)**: id, doctor_id, patient_name, gender, age, diagnosis_id (外键), visit_date, status (Draft/Submitted/Returned/Completed), create_time, update_time
- **VisitDrug (就诊处方明细表)**: id, visit_id, drug_id, quantity, amount
- **InventoryCheckTask (盘点任务表)**: id, month (例如 "2023-10"), status (Pending/Completed), create_time, complete_time
- **InventoryCheckDetail (盘点明细表)**: id, task_id, drug_id, system_stock (理论库存), actual_stock (实盘数量), discrepancy (差异), remark
- **OperationLog (操作日志表)**: id, user_id, action, timestamp
- **MonthlyStats (月度统计表)**: 用于持久化生成的月度报表数据

## 7. 非功能性需求
- **数据一致性**: 发药操作需使用数据库事务，确保库存扣减与记录状态更新同步。
- **实时性**: 药师端需轮询或通过WebSocket接收新处方通知。
- **易用性**: 医师录入界面应支持药品名称简拼搜索。
- **交互体验**: 各统计模块需支持月份筛选器，方便用户切换查看不同月份的数据。
