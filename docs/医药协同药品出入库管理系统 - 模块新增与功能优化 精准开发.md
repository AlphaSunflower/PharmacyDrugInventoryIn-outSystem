# 医药协同药品出入库管理系统 \- 模块新增与功能优化

核心要求：基于现有系统代码开发，**严格复用库存盘点模块的 UI 样式、交互逻辑、业务流程**，权限精准控制，数据校验严谨，生产厂家字段全局联动统一

---

## 一、整体需求总览

本次更新包含**4 大核心工作**：

1. 全新开发：采购计划模块

2. 全新开发：采购计划 Excel 导出功能

3. 功能优化：药品购进模块（新增生产厂家字段及联动逻辑）

4. 数据库 \+ 全局落地：新增`生产厂家`字段至【药品批次表】，覆盖采购计划 / 药品购进 / 药品管理初始批次

---

## 二、模块 1：全新开发【采购计划模块】

### 1\. 权限控制

仅允许角色：**ROOT、药师** 访问、编辑、操作

### 2\. 模块核心字段（自动填充 / 手动填写规则）

|字段名称|填充方式|严格执行规则|
|---|---|---|
|药品名称|系统自动导入|调取系统现有药品基础数据|
|规格|系统自动导入|匹配对应药品基础数据|
|单位|系统自动导入|匹配对应药品基础数据|
|进货价|系统自动导入|**强制获取对应药品最新批次的进货价**|
|生产厂家|自动填充 \+ 手动编辑|默认取药品最新批次厂家；无历史数据则为空；支持弹窗选历史厂家 / 自定义填写|
|数量|药师**手动必填**|采购计划唯一必填字段，仅药师可填写|

### 3\. UI 设计（1:1 复刻库存盘点模块）

#### 关键功能组件

1. 月份选择器：指定需要生成 / 查询的采购计划月份

2. 生成 / 获取当月采购计划按钮：模块核心触发按钮

3. 完成本次采购计划按钮：**仅生成计划 \+ 必填项填写完成后可用**

4. 删除任务按钮：生成计划后显示，一键清空当月所有填写数据

5. 重新修改按钮：对已完成的采购计划解锁编辑权限

6. 数据表格：分页展示所有字段，支持单元格点击编辑、实时更新

7. 未填写筛选按钮：新增按钮，过滤**去除空格后为空**的未填数据，快速查漏

### 4\. 宏观业务流程

1. 选择指定月份

2. 生成 / 获取对应月份的采购计划

3. 药师填写所有药品的计划数量（必填），可编辑生产厂家

4. 点击【未填写】筛选，补全遗漏数据

5. 所有必填项无空值 → 点击【完成本次采购计划】

6. 需重写数据 → 先获取计划，再点击【删除任务】

7. 已完成计划需修改 → 点击【重新修改】（小改动）；大批量改动直接删除任务重填

### 5\. 微观精准业务规则

1. 生成 / 获取逻辑：数据库无当月计划→自动生成；有则直接加载

2. 生产厂家交互：默认展示最新批次厂家 → 点击弹出窗口 → 按**时间新→旧**展示所有历史厂家 → 支持选中 / 自定义填写

3. 未填写筛选：**先去除空格**，再判断字段是否为空 / 空字符串，精准过滤遗漏项

4. 完成校验：**先去除空格**，所有「数量」字段非空才可完成，否则禁止并提示

5. 状态锁定：已完成的采购计划默认禁止编辑，需点击【重新修改】解锁

6. 删除规则：仅已生成 / 获取的计划可删除，清空当月全部数据

## 三、模块 2：全新开发【采购计划导出（Excel）】功能

### 1\. 权限控制

允许角色：**ROOT、药师、医师**

### 2\. 功能位置

集成在系统【导出报表】模块，**新增独立 Tab 页**

### 3\. 核心业务逻辑

1. 数据来源：根据月份选择器，查询**已完成状态**的当月采购计划

2. 拦截规则：计划未完成 → 弹出提示：「请先完成当月采购计划后再导出」

3. 导出内容：包含采购计划模块**所有字段 \+ 完整业务数据**

4. 导出格式：标准 Excel 文件

---

## 四、模块 3：【药品购进模块】全量优化

### 1\. 宏观优化点

1. 新增购进登记：新增`生产厂家`填写项

2. 购进记录列表：新增`生产厂家`展示列

3. 药品管理→药品详情表格：新增`生产厂家`展示列

### 2\. 微观精准业务规则

1. 新增购进登记・生产厂家联动：

    - 选中药品后，**自动填充最新批次的生产厂家**；无数据则为空

    - 输入框聚焦 → 弹窗展示该药品**所有历史厂家（新→旧排序）**

    - 支持选择历史厂家 / 点击自定义手动填写

2. 数据同步逻辑：

    - 待提交表格：实时展示填写的生产厂家

    - 提交新增记录后：**新药品批次的生产厂家同步为填写值**

---

## 五、全局落地：生产厂家字段新增（数据库 \+ 全模块）

### 1\. 数据库修改

在系统核心表【药品批次表】中 **新增 ****`生产厂家`**** 字段**

### 2\. 全模块覆盖落地

1. 采购计划模块：新增`生产厂家`字段（已定义）

2. 药品购进模块：新增购进登记填写`生产厂家`（已定义）

3. 药品管理模块：**创建药品初始批次**时，必须填写`生产厂家`

---

## 六、全局通用校验规则（所有模块强制生效）

1. 空值判断：所有字段校验**先去除空格**，再判断是否为空 / 空字符串

2. 数据联动：`生产厂家` 始终优先调取药品**最新批次**数据

3. 状态互斥：未生成采购计划 → 禁止完成 / 删除 / 修改；未完成计划 → 禁止导出
